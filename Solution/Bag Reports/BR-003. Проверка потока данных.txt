Название: Отсутствие сообщения об ошибке при вводе потока данных в невалидном формате

ID: BR-003

Приоритет: Средний.
Серьезность: Незначительная.

Повторяемость: Всегда.

Описание:
При вводе потока данных в невалидном формате (2 ввода "Enter" после значения конечной даты) программа завершается без каких-либо сообщений об ошибке, не предоставляя информации пользователю о неверно введенных данных.

Шаги воспроизведения:
1. Запустить программу.
2. Ввести данные в невалидном формате сразу в несколько переменных, например: "01.01.1900\n12.12.2000\n\n".
3. Проверить результат.

Фактический результат:
Программа завершается без сообщения об ошибке.

Ожидаемый результат:
Программа должна выдавать сообщение об ошибке при вводе данных в невалидном формате, информируя пользователя о неверно введенных данных.

Рекомендации:
Проанализировать и проверить логику обработки данных в невалидном формате в программе, обеспечить корректное предоставление сообщений об ошибке в таких ситуациях.

Воспроизводимое значение:
{
  "Initial_value": "01.01.1900\\n12.12.2000\\n\\n",
  "Result": "*Message* Данные переданы неверно. Попробуйте еще раз"
}
